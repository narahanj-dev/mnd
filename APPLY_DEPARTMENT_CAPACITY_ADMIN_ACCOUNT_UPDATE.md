# 부서별 기준 음영·관리자 계정 생성 방식 적용 안내

## 반영 내용

1. 달력의 휴가, 외박, 주말외출, 평일외출 인원을 사용자 중복 없이 합산합니다.
2. 각 부서의 활성 사용자 수를 기준으로 평일은 25% 초과, 토요일·일요일은 35% 초과 시 해당 날짜를 빨간색으로 표시합니다.
3. 관리자가 전체 부서를 조회할 때는 5개 부서를 각각 계산하며, 한 부서라도 기준을 초과하면 날짜가 빨간색으로 표시됩니다.
4. 로그인 화면의 회원가입 신청 버튼을 제거했습니다. 기존 회원가입 주소와 API도 사용할 수 없습니다.
5. 신규 계정은 관리자 로그인 후 `사용자 관리` → `+ 사용자 생성`에서 생성합니다.
6. 비밀번호 초기화 완료 후 아이디와 30분 유효 임시 비밀번호가 별도 팝업으로 표시되며 복사할 수 있습니다.

## 적용 순서

이번 수정은 데이터베이스 구조 변경이 없어 Supabase SQL 실행이 필요하지 않습니다.

```powershell
cd D:\leave-calendar
npm install
npm run build
git add .
git commit -m "부서별 일정 기준 음영과 관리자 계정 발급 방식 적용"
git push origin master
```

GitHub 기본 브랜치가 `main`이면 마지막 명령을 다음과 같이 사용합니다.

```powershell
git push origin main
```

## 확인 항목

- 평일 한 부서의 4개 항목 사용자 합계가 부서원 25%를 초과하면 날짜가 빨간색인지 확인
- 토요일 또는 일요일에 35%를 초과하면 날짜가 빨간색인지 확인
- 동일 사용자가 같은 날 여러 일정을 등록해도 한 명으로 계산되는지 확인
- 로그인 화면에 회원가입 신청 버튼이 없는지 확인
- `/signup-request` 접속 시 로그인 화면으로 돌아오는지 확인
- 관리자 사용자 관리에서 계정 생성이 가능한지 확인
- 비밀번호 초기화 성공 후 임시 비밀번호 팝업과 복사 버튼이 표시되는지 확인
