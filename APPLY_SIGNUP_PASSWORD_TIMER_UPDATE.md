# 회원가입 비밀번호 설정·300초 자동 로그아웃 적용방법

## 반영 내용

1. 회원가입 신청 화면에 비밀번호와 비밀번호 확인 입력란을 추가했습니다.
2. 기존 비밀번호 보안정책인 9자 이상, 영문 대문자·소문자·숫자·특수문자 중 3종류 이상 규칙을 회원가입에도 적용합니다.
3. 신청 비밀번호는 서버에서 AES-256-GCM으로 암호화하여 가입신청 DB에 임시 저장합니다.
4. 관리자 승인 시 비밀번호를 다시 입력하지 않고 신청자가 설정한 비밀번호로 계정을 생성합니다.
5. 승인 또는 거절 후 가입신청 행이 삭제되므로 임시 비밀번호 암호문도 함께 삭제됩니다.
6. 승인된 사용자는 신청 때 설정한 비밀번호로 바로 로그인하며 최초 로그인 강제 변경창은 뜨지 않습니다.
7. 로그인 후 상단에 05:00 타이머가 표시됩니다.
8. 다른 페이지로 이동하거나 브라우저를 새로고침하면 타이머가 다시 05:00으로 초기화됩니다.
9. 타이머가 00:00이 되면 자동 로그아웃되고 로그인 화면에 만료 안내가 표시됩니다.

## 1단계: 소스 덮어쓰기

수정 압축파일의 전체 내용을 기존 프로젝트 폴더에 덮어씁니다.

```powershell
cd D:\leave-calendar
npm install
```

## 2단계: Supabase SQL 실행

Supabase에서 **SQL Editor → New query**를 누르고 아래 파일 내용을 전부 붙여넣어 실행합니다.

```text
supabase/migration_20260718_signup_password_session_timeout.sql
```

성공하면 `signup_requests` 테이블에 `requested_password` 컬럼이 추가됩니다.

## 3단계: 검사

```powershell
npm run lint
npm run build
```

## 4단계: GitHub 배포

```powershell
git add .
git commit -m "회원가입 비밀번호 설정과 5분 자동 로그아웃 적용"
git push
```

Vercel을 사용 중이면 GitHub 푸시 후 자동 배포가 완료될 때까지 배포 화면에서 오류가 없는지 확인합니다.

## 5단계: 기능 확인

1. 회원가입에서 비밀번호와 비밀번호 확인이 일치하지 않으면 신청되지 않는지 확인합니다.
2. 9자 미만 또는 보안정책을 충족하지 않는 비밀번호가 거절되는지 확인합니다.
3. 정상 신청 후 관리자 가입신청 화면에서 비밀번호 입력창 없이 승인되는지 확인합니다.
4. 승인된 계정이 신청자가 설정한 비밀번호로 로그인되는지 확인합니다.
5. 로그인 후 상단 타이머가 05:00부터 감소하는지 확인합니다.
6. 달력에서 사용현황 등 다른 메뉴로 이동할 때 05:00으로 초기화되는지 확인합니다.
7. 새로고침할 때 05:00으로 초기화되는지 확인합니다.
8. 아무 페이지 이동 없이 5분이 지나면 자동 로그아웃되는지 확인합니다.

## 기존 대기 중 가입신청 주의

이번 업데이트 전에 접수된 가입신청에는 설정된 비밀번호가 없습니다. 해당 신청을 승인하면 재신청 안내 오류가 표시됩니다. 기존 신청은 거절 처리한 뒤 사용자가 새 회원가입 화면에서 다시 신청하도록 안내하세요.

## 보안키 주의

`PII_ENCRYPTION_KEY`는 기존 개인정보 암호화에 사용하던 값과 반드시 동일해야 합니다. 키를 변경하거나 분실하면 신청 비밀번호를 포함한 암호화 데이터를 복호화할 수 없습니다.
