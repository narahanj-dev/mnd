# 일반사용자 비밀번호 변경 기능 적용 방법

## 반영 내용

- 일반사용자가 `사용자 관리` 화면에서 본인 계정만 조회하는 기존 구조를 유지합니다.
- 본인 계정의 관리 항목에 다음 버튼이 표시됩니다.
  - 아이디 변경
  - 비밀번호 변경
- 비밀번호 변경 시 다음 내용을 입력합니다.
  - 현재 비밀번호
  - 새 비밀번호
  - 새 비밀번호 확인
- 현재 비밀번호가 맞아야 변경할 수 있습니다.
- 기존 비밀번호 보안 규칙과 최근 비밀번호 5개 재사용 제한을 그대로 적용합니다.
- 관리자와 부서관리자의 `비밀번호 초기화` 기능은 기존과 동일합니다.
- 임시 비밀번호 또는 6개월 경과로 강제 변경하는 화면도 기존과 동일하게 작동합니다.

## 수정된 파일

```text
components/admin/UserManagement.tsx
app/api/auth/change-password/route.ts
```

## Supabase 적용 여부

이번 수정은 데이터베이스 구조를 변경하지 않으므로 Supabase SQL을 실행할 필요가 없습니다.

## 적용 순서

1. 기존 프로젝트의 `.env.local` 파일을 별도로 보관합니다.
2. 수정 압축파일을 해제합니다.
3. 압축을 푼 파일을 기존 프로젝트 폴더에 전체 덮어씁니다.
4. 기존 `.env.local` 파일이 유지됐는지 확인합니다.
5. PowerShell에서 다음 명령을 실행합니다.

```powershell
cd D:\leave-calendar
npm install
npm run build
git add .
git commit -m "일반사용자 비밀번호 변경 기능 추가"
git push origin master
```

현재 브랜치가 `main`이면 마지막 명령을 다음처럼 실행합니다.

```powershell
git push origin main
```

## 적용 확인

1. 일반사용자 계정으로 로그인합니다.
2. 상단 메뉴에서 `사용자 관리`를 선택합니다.
3. 본인 계정의 관리 항목에서 `비밀번호 변경`을 선택합니다.
4. 현재 비밀번호와 새 비밀번호를 입력합니다.
5. 변경 완료 후 로그아웃합니다.
6. 변경한 새 비밀번호로 다시 로그인되는지 확인합니다.

## 비밀번호 규칙

- 9자 이상
- 영문 대문자, 영문 소문자, 숫자, 특수문자 중 3종류 이상 포함
- 아이디 또는 이름 포함 금지
- 연속 숫자와 추측하기 쉬운 문자열 금지
- 최근 사용한 비밀번호 5개 재사용 금지
