# 일반사용자 본인 사용자 관리 적용 안내

## 반영 내용

- 일반사용자 메뉴에 `사용자 관리`가 표시됩니다.
- 일반사용자는 사용자 관리 화면에서 본인 계정 한 건만 확인할 수 있습니다.
- 일반사용자는 본인 아이디만 변경할 수 있습니다.
- 일반사용자는 다른 사용자를 조회하거나 수정할 수 없습니다.
- 일반사용자의 부서·권한 변경, 비밀번호 초기화, 계정 삭제 기능은 차단됩니다.
- 관리자와 부서관리자의 기존 사용자 관리 권한은 그대로 유지됩니다.

## 적용 방법

1. 수정 압축파일의 전체 내용을 기존 프로젝트 폴더에 덮어씁니다.
2. 기존 `.env.local`과 `.git` 폴더는 삭제하지 않습니다.
3. 프로젝트 폴더에서 다음 명령을 실행합니다.

```powershell
Remove-Item -Recurse -Force .next -ErrorAction SilentlyContinue
npm install
npm run lint
npm run build
```

4. 로컬 테스트를 실행합니다.

```powershell
npm run dev
```

5. 일반사용자로 로그인하여 다음 사항을 확인합니다.

- 상단 메뉴에 `사용자 관리`가 나타나는지
- 본인 계정만 보이는지
- 아이디 변경만 가능한지
- 부서·권한 변경, 비밀번호 초기화, 삭제 버튼이 없는지

6. 확인 후 GitHub에 반영합니다.

```powershell
git add .
git commit -m "일반사용자 본인 사용자 관리 허용"
git push
```
