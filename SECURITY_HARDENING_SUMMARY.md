# 보안 강화 완료 요약

## 반영한 핵심 개선

1. 달력 API는 `approved` 일정만 DB 조회 단계에서 가져오도록 변경했습니다.
2. 타인의 사용자 ID와 일정 ID를 서버 HMAC 기반 식별자로 치환했습니다.
3. 달력 응답에서 설명, 공개메모, 관리자메모, 거절사유, 승인자, 생성·수정시각을 제외했습니다.
4. 관리자·부서관리자의 일정·쪽지·비밀번호 관련 API에 MFA AAL2 검사를 추가했습니다.
5. Supabase 인증 쿠키를 `HttpOnly`로 바꾸고 브라우저 Supabase 클라이언트를 제거했습니다.
6. MFA 등록·검증을 `/api/auth/mfa` 서버 API로 이동했습니다.
7. 일정 승인과 일정 변경 승인을 PostgreSQL 트랜잭션 함수로 변경했습니다.
8. 달력 조회 범위는 최대 1,500일, 단일 일정은 최대 366일로 제한했습니다.
9. 세션은 300초 유휴 만료와 별도로 최대 8시간 절대 만료를 적용했습니다.
10. CSRF 검증 기준을 고정 `APP_ORIGIN`으로 변경했습니다.
11. Vercel·Cloudflare 등 신뢰 가능한 프록시 IP 헤더를 우선 사용하도록 변경했습니다.
12. PII 암호화키는 32바이트 Base64 또는 64자리 16진수만 허용하도록 강화했습니다.
13. 임시비밀번호 이력 저장 실패를 성공으로 무시하지 않도록 변경했습니다.
14. 관리자 설정 변경 실패 시 기존 이메일·비밀번호·메타데이터 복구 로직을 강화했습니다.
15. 민감 응답에 `Cache-Control: no-store`를 추가했습니다.
16. GET 쪽지 조회 중 삭제 작업을 수행하던 부작용을 제거했습니다.
17. 쪽지·속도제한·감사로그 보존 정리 함수를 추가했습니다.
